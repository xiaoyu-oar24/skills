# z-paging-best-practices 触发评测集

> 评估驱动开发：修改 `description` 或触发条件后，用以下提示词回归验证路由行为。

## 应触发（Expected：激活）

1. "帮我写一个商品列表页，要能下拉刷新和上滑加载"
2. "这个页面的导航栏把列表第一行挡住了"（uni-app + z-paging 项目）
3. "列表页底部的操作按钮被内容顶出去了，帮我修一下"

## 不应触发（Expected：跳过）

1. "帮我写一个登录表单页"（非分页列表场景）
2. "这个 React 项目的表格分页怎么做？"（非 uni-app 技术栈）
3. "帮我改一下首页轮播图"
